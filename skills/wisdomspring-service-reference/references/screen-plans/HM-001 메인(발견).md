# HM-001 메인(발견)

## 기본 정보

| 항목 | 내용 |
| --- | --- |
| 공통 화면 ID | HM-001 |
| 모바일 화면 ID | M-HM-001 |
| 웹 화면 ID | W-HM-001 |
| 화면명 | 메인(발견) |
| 화면 유형 | 페이지 |

## 화면 개요

위즈덤스프링 홈의 메인 발견 화면이다. 오늘의 질문, 테마 카드, 시청 중인 영상, 오늘의 한 문장, 큐레이션 영역, 모든 테마 탐색 영역을 통해 사용자가 주요 콘텐츠와 테마를 탐색하고 진입할 수 있도록 구성한다.

## 상세 설명

### 1. 오늘의 질문

- `Daily 오늘의 질문`을 노출한다.
- 타이틀은 `오늘의 질문`으로 고정한다.
- 질문 텍스트는 실제 오늘의 질문 데이터로 노출한다.
- 예시: `자본주의의 본질은 거래 그 자체일까. 아니면 직거래를 가능하게 하는 신뢰일까?`
- 질문 텍스트 길이는 최대 50자까지로 제한한다.
- 50자 제한은 Admin에서 관리한다.
- 프론트에서는 영역을 초과할 경우 말줄임 처리한다.
- `답변하기` 버튼을 제공한다.
- `답변하기` 버튼 영역 클릭 시 디바이스별로 아래 화면을 호출한다.

| 디바이스 | 호출 화면 | 방식 |
| --- | --- | --- |
| 모바일 | M-FD-B030 | 바텀시트 |
| 웹 | W-FD-M030 | 모달 오버레이 |

- 이후 답변 작성 액션은 별도 페이지 정의를 따른다.

### 2. 테마 카드

- 테마(LV1) 리스트를 카드 형식으로 노출한다.
- 예시: `괴테 / 논어 / 르네상스 / 돈키호테`
- 카드 리스트는 슬라이딩 방식으로 탐색한다.
- 최신 테마 카드가 항상 중앙에 위치해야 한다.
- 첫 화면 진입 시에는 좌측 카드 없이 중앙 카드부터 우측으로 리스트가 나열된다.
- PC에서는 기본적으로 카드 3개가 노출된다.
- 디바이스 사이즈에 맞춰 카드 섹션 개수는 가변적으로 줄어든다.
- 양쪽 끝 카드에서는 해당 방향의 화살표를 노출하지 않는다.
- 테마 카드 썸네일은 오픈 날짜에 따라 변경된다.
- 썸네일별 시스템 문구를 노출한다.
- 시스템 문구는 Admin에서 작성한다.
- 시스템 문구는 최대 20자까지 입력 가능하다.
- Admin에서 `<br>` 줄바꿈을 허용한다.
- 시스템 문구가 3줄을 초과할 경우 텍스트가 잘리도록 처리한다.
- 오픈 날짜에 따라 신규 테마를 메인에 노출한다.
- 신규 테마에는 `NEW` 배지를 표시한다.
- 배지에는 서브테마 오픈 현황을 `N개 중 N개 오픈` 형식으로 표시한다.
- 서브테마 오픈 현황 기준은 각 서브테마 오픈일이다.

#### 모아보기 액션

- `모아보기` 클릭 시 전체 테마 카드 리스트를 노출한다.

| 디바이스 | 호출 화면 | 방식 |
| --- | --- | --- |
| 모바일 | M-HM-B002 | 바텀시트 |
| 웹 | W-HM-P002 | 팝업 |

### 3. 내가 시청중인 영상

- 시청 중인 영상은 `TALK`, `BOOK` 유형만 노출한다.
- 전체 항목은 슬라이드 방식으로 노출한다.
- 노출 항목은 아래와 같다.
  - 테마명
  - 서브테마 유형(`TALK`, `BOOK`, `ESSAY`)
  - 콘텐츠 썸네일
  - 콘텐츠 제목
- 썸네일 하단에는 프로그레스 바를 노출한다.
- 영상 총 재생시간 정보를 함께 제공한다.

### 4. 오늘의 한 문장

- 오늘의 콘텐츠에서 추출한 오늘의 한 문장을 모아 명언 카드 형태로 제공한다.
- 각 카드는 관련 콘텐츠로 이동할 수 있어야 한다.
- 노출 항목은 아래와 같다.
  - 오늘의 한 문장
  - 테마명
  - 콘텐츠명
  - 콘텐츠 바로가기 CTA
- 오늘의 한 문장은 최대 35자까지 노출한다.
- PC에서는 최대 2줄까지 노출한다.
- 모바일에서는 최대 3줄까지 노출한다.
- 웹에서는 썸네일과 시간 정보를 함께 노출한다.
- 시간 정보는 영상일 경우 재생시간, 아티클일 경우 분량 분수로 표시한다.

#### 바로가기 액션

- `바로가기` 클릭 시 콘텐츠 유형에 따라 아래 학습화면으로 이동한다.

| 콘텐츠 유형 | 이동 화면 |
| --- | --- |
| 영상 | M-HM-020 / W-HM-020 |
| 아티클 | M-HM-021 / W-HM-021 |

- 카드는 최대 20개까지 노출한다.
- 슬라이드 방식으로 탐색한다.
- `전체보기`는 제공하지 않는다.

### 5. 큐레이션 항목

- `추천`, `인기`, `최신`, `찜 많은` 등 큐레이션 성격의 항목을 노출한다.
- `TALK`, `BOOK`, `ESSAY` 유형별 큐레이션 영역으로 구성한다.

#### 5-1. 시청률이 높은 대담

- `TALK` 콘텐츠를 대상으로 노출한다.
- 클릭 시 해당 콘텐츠 학습화면으로 이동한다.
- 로직은 당일 기준 최근 1주일간 재생 이력이 가장 많은 `TALK` 콘텐츠 순으로 구성한다.
- 조회수가 아닌 재생이력을 기준으로 한다.
- 노출 항목은 아래와 같다.
  - 재생 버튼
  - 썸네일
  - 강사 2명 정보(프로필 이미지, 강사명)
  - 재생시간
  - 콘텐츠 제목
  - 테마명
- 최대 20개까지 노출한다.
- `전체보기`는 제공하지 않는다.

#### 5-2. 오늘의 추천 북러닝

- `BOOK` 서브테마를 노출한다.
- 클릭 시 해당 테마 상세 페이지로 이동한다.
- 이동 후 클릭한 북러닝 서브테마 섹션으로 스크롤 이동한다.
- 로직은 최대 10개 랜덤 노출이다.
- 노출 항목은 아래와 같다.
  - 썸네일
  - 서브테마명
  - 강사 정보(프로필 이미지, 강사명)
  - 영상 개수
  - 테마명

#### 5-3. 가장 많이 읽은 에세이

- `ESSAY` 콘텐츠를 대상으로 노출한다.
- 클릭 시 해당 에세이 콘텐츠 학습화면으로 이동한다.
- 노출 항목은 아래와 같다.
  - 테마명
  - 에세이 제목
  - 본문 미리보기
  - 저자명
  - 리딩 시간
- 최대 20개까지 노출한다.
- 찜(저장) 개수는 노출하지 않는다.
- 노출 로직은 별도 정의 필요(TBD).

### 6. 모든 테마 탐색

- 테마 카드 리스트를 노출한다.
- `전체보기` 선택 시 아래 화면을 호출한다.

| 디바이스 | 호출 화면 |
| --- | --- |
| 모바일 | M-HM-B002 |
| 웹 | W-HM-P002 |

- 테마 선택 시 카드 전체 영역을 버튼으로 동작시킨다.
- 테마 제목 길이는 백엔드에서 1줄을 넘지 않도록 제한한다.
- 각 테마 선택 시 아래 테마 상세 페이지로 이동한다.

| 디바이스 | 이동 화면 |
| --- | --- |
| 모바일 | M-HM-010 |
| 웹 | W-HM-010 |

## 비고

- `내가 시청중인 영상` 영역 설명에는 노출 유형이 `TALK`, `BOOK`만이라고 정의되어 있으나, 구성요소 예시에는 `ESSAY`가 함께 기재되어 있다. 실제 노출 대상 유형은 확인이 필요하다.
- `가장 많이 읽은 에세이`의 정렬 및 선정 로직은 추가 정의가 필요하다.
