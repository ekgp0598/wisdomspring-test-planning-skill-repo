### 기대결과 묶음 작성 규칙

각 테스트 케이스의 기대결과는 가능하면 아래 항목으로 분해해서 적는다.

- 정상:
- 예외:
- 상태변화:
- 데이터 반영:
- 플랫폼 차이:
- 정책 연동:

모든 항목을 항상 다 채울 필요는 없지만, 해당 흐름에 의미 있는 항목은 비워두지 않는다.

## 5. 발견사항

For each issue, capture:

- 제목
- 심각도
- 재현 절차
- 기대결과
- 실제결과
- 영향 범위
- 관련 화면 ID or 파일

## 6. 오픈 이슈

- 명세가 비어 있거나 충돌하는 항목
- 환경이나 데이터가 없어 확인하지 못한 항목
- 정상 결과는 확인됐지만 예외 결과는 미검증인 항목
